# URL → Markdown 변환기

## 개요

웹 페이지 URL을 본문 Markdown으로 변환해 사용자가 바로 LLM(ChatGPT, Claude)으로 넘기거나 복사·다운로드할 수 있게 한다. 사용자는 URL을 붙여넣고 "변환"을 누른 뒤, 결과 헤더와 렌더링된 Markdown을 보고, 프롬프트를 골라 LLM에 전달하거나 그대로 내보낸다.

## 범위

### 포함

- URL 입력 → 본문 Markdown 변환 (한 번에 하나)
- 변환 결과의 헤더(제목·저자) + Markdown 렌더링
- 복사 / `.md` 다운로드 / ChatGPT로 열기 / Claude로 열기
- 프롬프트 프리셋 3종 (요약 / 한국어 번역 / 쉽게 설명) + "직접 입력" 1회용 텍스트
- 입력 지우기(URL과 결과 모두 리셋)
- 다크모드 토글
- 입력/네트워크/추출 오류의 인라인 표시

### 제외

- **변환 이력·저장** — 결과는 페이지 새로고침 시 소실. MVP에서 영속성은 다음 단계로 미룬다.
- **여러 URL 일괄 변환** — 한 번에 하나만. 일괄 처리는 UI 복잡도가 크다.
- **계정·인증·동기화** — 단일 페이지 익명 사용.
- **변환 결과 편집** — Markdown은 읽기 전용. 인라인 수정은 향후 과제.
- **모바일 최적화** — 데스크톱 우선. 모바일에서 동작하되 레이아웃 최적화는 별도 작업.
- **프롬프트 영속화** — 프리셋·직접 입력 내용은 매 세션 기본값으로 시작 (다크모드는 유지).

## 시나리오

### 1. 유효한 URL을 Markdown으로 변환한다

- **Given** — 빈 URL 입력란이 있는 초기 화면
- **When** — `https://example.com/article` 형태의 URL을 붙여넣고 "변환" 버튼을 누른다
- **Then** — 변환 중 상태가 잠시 보인 뒤, 결과 영역에 페이지 제목·저자 헤더와 렌더링된 Markdown이 나타난다. "복사", ".md 다운로드", "ChatGPT로 열기", "Claude로 열기" 액션 버튼이 함께 보인다.

성공 기준:
- [ ] 본문이 있는 일반 기사 URL → 결과 영역에 헤더(제목 텍스트)와 Markdown 변환 본문이 표시된다
- [ ] 변환이 진행되는 동안 "변환" 버튼은 비활성화되고 진행 중 표시가 보인다
- [ ] 변환이 끝나면 4개의 액션 버튼(복사, 다운로드, ChatGPT로 열기, Claude로 열기)이 결과 옆/아래에 표시된다

### 2. 잘못된 URL 포맷은 변환을 막는다

- **Given** — URL 입력란에 포커스가 있다
- **When** — `not a url` 같은 http(s) 스킴이 아닌 텍스트를 입력한다
- **Then** — "변환" 버튼은 비활성화 상태가 되고, 입력란 바로 아래에 "올바른 URL을 입력하세요" 형태의 인라인 메시지가 표시된다.

성공 기준:
- [ ] 빈 입력 → "변환" 버튼 비활성화, 에러 메시지는 표시되지 않는다(아직 시도하지 않음)
- [ ] `not a url` 입력 → "변환" 버튼 비활성화 + 입력란 아래 포맷 오류 메시지
- [ ] 유효한 `https://...` 입력 → 메시지 사라지고 "변환" 버튼 활성화

### 3. 가져오기·추출 실패는 사용자에게 알리고 재시도를 허용한다

- **Given** — 유효한 형식이지만 접근할 수 없거나 본문 추출이 실패하는 URL이 입력되어 있다
- **When** — "변환" 버튼을 누른다
- **Then** — 진행 중 상태가 끝난 뒤, 결과 영역은 비어 있고 입력란 아래에 실패 원인을 알리는 메시지(예: "페이지를 가져올 수 없습니다" / "본문을 찾지 못했습니다")가 인라인으로 표시된다. URL은 입력란에 그대로 남아 사용자가 다시 시도하거나 수정할 수 있다.

성공 기준:
- [ ] 가져오기 실패 → 입력란 아래 네트워크 오류 메시지, 결과 영역은 비어 있음
- [ ] 추출 실패(본문 없음) → 입력란 아래 추출 오류 메시지, 결과 영역은 비어 있음
- [ ] 실패 후 URL 값은 입력란에 보존된다
- [ ] 실패 후 "변환" 버튼은 다시 활성화되어 재시도 가능

### 4. 메타데이터(제목·저자)가 없는 페이지도 변환된다

- **Given** — 유효하지만 제목이나 저자 메타데이터를 찾을 수 없는 URL
- **When** — 변환에 성공한다
- **Then** — 헤더는 추출된 항목만 표시한다. 제목이 없으면 입력한 원본 URL을 제목 자리에 표시하고, 저자가 없으면 저자 줄을 생략한다. Markdown 본문은 정상적으로 렌더링된다.

성공 기준:
- [ ] 제목 없음 + 저자 있음 → 헤더에 원본 URL과 저자
- [ ] 제목 있음 + 저자 없음 → 헤더에 제목만, 저자 줄 없음
- [ ] 제목 없음 + 저자 없음 → 헤더에 원본 URL만

### 5. 변환된 Markdown을 클립보드에 복사한다

- **Given** — 변환에 성공한 결과 화면, 프롬프트는 어느 것이 선택되어 있어도 무방
- **When** — "복사" 버튼을 누른다
- **Then** — 사용자의 클립보드에 변환된 Markdown 본문만(프롬프트 미포함) 담기고, 복사 완료를 알리는 짧은 피드백(예: 버튼 텍스트 변화 또는 토스트)이 표시된다.

성공 기준:
- [ ] 복사 직후 클립보드 내용 = 변환된 Markdown (프리셋·직접 입력 선택 여부와 무관)
- [ ] 복사 완료를 알리는 시각 피드백이 잠깐 보였다 사라진다

### 6. 변환된 Markdown을 `.md` 파일로 다운로드한다

- **Given** — 변환에 성공한 결과 화면
- **When** — ".md 다운로드" 버튼을 누른다
- **Then** — 브라우저가 `.md` 파일을 내려받는다. 파일명은 추출된 페이지 제목을 기반으로 정해지고, 제목이 없으면 원본 URL의 도메인+경로를 사용한다. 파일 내용은 변환된 Markdown 본문만 담는다(프롬프트 미포함).

성공 기준:
- [ ] 제목 "How React Works" → `How-React-Works.md` 같은 파일명으로 다운로드
- [ ] 제목 없음, URL `https://example.com/blog/post` → `example.com-blog-post.md` 같은 파일명
- [ ] 다운로드된 파일을 열면 본문은 결과 화면에 렌더링된 Markdown과 동일하다

### 7. ChatGPT/Claude로 열기는 전달 방식을 선택하게 한다

- **Given** — 변환에 성공한 결과 화면. "요약해줘" 프리셋이 선택되어 있음
- **When** — "ChatGPT로 열기" 버튼을 누른다
- **Then** — "클립보드 복사 후 열기" / "URL 파라미터로 전달" 두 옵션을 보여주는 안내가 뜬다. 사용자가 한쪽을 선택하면, 그 방식에 맞춰 새 탭에서 ChatGPT(또는 Claude)가 열리거나 클립보드에 텍스트가 담긴 채 채팅창이 열린다. 어느 방식이든 전달되는 텍스트는 `요약해줘\n\n<Markdown 본문>` 형태다.

성공 기준:
- [ ] 버튼 클릭 → "클립보드 복사 후 열기" / "URL 파라미터로 전달" 두 선택지가 보이는 안내
- [ ] "클립보드 복사 후 열기" 선택 → 클립보드에 `프롬프트\n\nMarkdown` 형태의 텍스트가 담기고 새 탭이 ChatGPT/Claude 메인으로 열린다
- [ ] "URL 파라미터로 전달" 선택 → 새 탭이 같은 텍스트를 쿼리로 담은 URL로 열린다
- [ ] 프롬프트 미선택 상태에서는 전달 텍스트가 Markdown 본문만 (앞 줄바꿈 없음)
- [ ] "Claude로 열기"도 같은 흐름으로 동작 (대상 호스트만 다름)

### 8. 프롬프트 프리셋을 선택하면 LLM 액션에 반영된다

- **Given** — 변환에 성공한 결과 화면, 프롬프트는 선택되지 않은 상태
- **When** — 사용자가 "한국어로 번역해줘" 프리셋을 선택한 뒤 "Claude로 열기" → "URL 파라미터로 전달"을 선택한다
- **Then** — 새 탭에서 열리는 URL 파라미터에 전달되는 텍스트는 `한국어로 번역해줘\n\n<Markdown 본문>`이다.

성공 기준:
- [ ] 프롬프트 프리셋 3종 ("요약해줘", "한국어로 번역해줘", "쉽게 설명해줘") 중 하나를 선택할 수 있고 한 번에 하나만 선택된다
- [ ] 선택된 프리셋이 전달 텍스트의 첫 줄로 들어가고, 그 뒤에 빈 줄, 그 뒤에 Markdown 본문이 이어진다
- [ ] 선택을 해제할 수 있고, 해제 시 LLM 액션은 Markdown 본문만 전달한다

### 9. "직접 입력" 모드는 1회용 프롬프트를 받는다

- **Given** — 변환에 성공한 결과 화면, 프롬프트 프리셋 옆에 "직접 입력" 옵션이 있다
- **When** — "직접 입력"을 선택하고 텍스트 인풋에 `이 글의 핵심 주장을 3개로 정리해줘`라고 적은 뒤 "ChatGPT로 열기"를 누른다
- **Then** — 전달되는 텍스트는 `이 글의 핵심 주장을 3개로 정리해줘\n\n<Markdown 본문>`이다. 직접 입력 내용은 어디에도 저장되지 않으며, 페이지를 새로고침하거나 "지우기"를 누르면 사라진다.

성공 기준:
- [ ] "직접 입력" 선택 → 텍스트 인풋이 표시되고 프리셋 선택은 해제된다
- [ ] 텍스트 입력값이 전달 텍스트의 첫 줄로 들어간다
- [ ] 페이지 새로고침 후 다시 들어오면 직접 입력 내용은 비어 있다
- [ ] 직접 입력 텍스트가 비어 있는 상태에서 LLM 액션 → 프롬프트 없는 것과 동일(Markdown만 전달)

### 10. "지우기"는 URL과 변환 결과를 모두 초기화한다

- **Given** — URL이 입력되어 있고 변환 결과가 화면에 표시된 상태. 프롬프트도 선택되어 있을 수 있음
- **When** — "지우기" 버튼을 누른다
- **Then** — URL 입력란이 비고, 결과 영역이 사라지며, 인라인 에러 메시지도 모두 사라진다. 프롬프트 선택과 직접 입력 내용도 초기 상태로 돌아간다. 다크모드 상태는 그대로 유지된다.

성공 기준:
- [ ] 지우기 후 URL 입력란 = 빈 문자열, 결과 영역 미표시
- [ ] 지우기 후 인라인 에러 메시지가 있다면 사라진다
- [ ] 지우기 후 프롬프트 선택 해제, 직접 입력 내용 비움
- [ ] 다크모드 토글 상태는 지우기에 영향받지 않는다

### 11. 다크모드 토글은 새로고침 후에도 유지된다

- **Given** — 라이트 모드 상태의 초기 화면
- **When** — 다크모드 토글을 누르고 페이지를 새로고침한다
- **Then** — 새로고침된 화면도 다크모드로 유지된다. 다시 토글을 누르면 라이트 모드로 돌아가고, 또 새로고침해도 라이트 모드가 유지된다.

성공 기준:
- [ ] 다크모드로 전환 → 페이지 배경/텍스트가 다크 팔레트로 변경
- [ ] 새로고침 후에도 다크모드 유지
- [ ] 라이트로 되돌리고 새로고침 → 라이트 모드 유지

## 불변 규칙

### LLM 전달 텍스트 형식

"ChatGPT로 열기" / "Claude로 열기"가 전달하는 텍스트는 항상 다음 규칙을 따른다.

- 프롬프트가 선택되어 있으면 `<프롬프트 텍스트>\n\n<Markdown 본문>`
- 프롬프트가 없으면 `<Markdown 본문>` (앞 줄바꿈 없음)
- 전달 방식(클립보드 / URL 파라미터)이 달라도 텍스트 자체는 동일

### "복사" / "다운로드"는 항상 순수 Markdown

"복사" 버튼과 ".md 다운로드"는 프롬프트 선택 여부와 무관하게 변환된 Markdown 본문만 다룬다. 프롬프트는 LLM 전달 액션에만 영향을 준다.

### 인라인 에러 표시 위치

URL 포맷 오류, 가져오기 실패, 본문 추출 실패는 모두 URL 입력란 바로 아래에 인라인으로 표시한다. 결과 영역이나 별도 알림 영역에는 에러를 띄우지 않는다. 결과 영역은 성공한 변환 결과만 보여준다.

## 의존성

- **defuddle** — 본문 추출 라이브러리 (요구사항에 명시된 필수 의존성)
- **next-themes** — 다크모드 토글 영속성 (현재 프로젝트에 이미 설치되어 있음)

## 미결정 항목

- **프롬프트의 초기 기본값** — 매 세션 "선택 없음"으로 시작할지, 또는 첫 프리셋("요약해줘")이 선택된 상태로 시작할지 사용자에게 확인하지 못했다. 현재 spec은 "선택 없음"을 가정하고 작성했으며, 시나리오 8에서 사용자가 명시적으로 선택해야 LLM 액션에 반영된다고 했다.
- **"직접 입력"의 표시 위치 / 전환 방식** — 프리셋과 같은 묶음 안의 라디오 옵션인지, 별도 토글로 영역이 펼쳐지는지는 결정하지 못했다. 와이어프레임 단계에서 정한다.
