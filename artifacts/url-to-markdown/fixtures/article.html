How React Works

By Jane Doe

React renders user interfaces through a virtual representation of the DOM. When state changes, React reconciles the previous tree with the next tree and produces the smallest set of mutations required to bring the document in line.

Reconciliation

The reconciler walks the component tree top-down. Identical element types are diffed in place; differing types are unmounted and remounted. Keys disambiguate sibling order across renders.

  • Diff by type and key
  • Reuse hooks across renders
  • Schedule effects after commit

Commit Phase

After reconciliation produces a work-in-progress tree, the commit phase applies the changes synchronously and then flushes effects asynchronously in a separate microtask.
