인터랙션:

  • URL 입력란 포커스 → 사용자가 URL을 붙여넣는다

참고: 빈 입력은 인라인 에러를 띄우지 않는다 (아직 시도하지 않은 상태).

인터랙션:

  • URL 입력란은 그대로 노출 — 사용자가 다른 URL로 다시 변환할 수 있다

  • "직접 입력" 선택 시: 라디오 그룹 아래에 텍스트 인풋이 펼쳐진다 (시나리오 9 화면 예정)

  • LLM "열기" 클릭 → 전달 방식 다이얼로그 (시나리오 7)

레이아웃 결정: "직접 입력"은 프리셋과 같은 라디오 그룹의 4번째 옵션 (한 번에 하나만 선택된다는 규칙을 시각적으로 강제). 선택 시 그 아래로 텍스트 인풋이 펼쳐진다.

인터랙션:

전환: Result 화면 (변화 없음)

불변 규칙: 복사 내용은 항상 순수 Markdown 본문 — 프롬프트는 절대 포함되지 않는다.

Screen Notes

트리거: Result 화면에서 "ChatGPT로 열기" 또는 "Claude로 열기" 클릭

인터랙션:

  • 두 옵션 중 하나 선택 → 새 탭이 열리며 그 방식대로 텍스트 전달
  • "클립보드 복사 후 열기": 클립보드에 텍스트 담기 + 새 탭이 LLM 메인 페이지
  • "URL 파라미터로 전달": 텍스트를 쿼리에 담은 URL로 새 탭 열기
  • 배경 클릭 또는 "취소" → 다이얼로그 닫힘 (Result 화면으로 복귀)

전환:

  • 옵션 선택 후 → 다이얼로그 닫히고 Result 화면 유지 (새 탭이 별도로 열린 상태)
  • 취소 → Result 화면

불변 규칙: 전달 텍스트는 두 방식이 동일해야 함. 프롬프트가 없으면 Markdown만(앞 줄바꿈 없음). "Claude로 열기"도 같은 다이얼로그 — 대상 호스트만 다름.
